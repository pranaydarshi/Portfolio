# Local Run Guide — Portfolio Backend + Frontend
# Darshi Sai Pranay | ASP.NET Core 8 + React

=============================================================
 PREREQUISITES
=============================================================

Make sure these are installed before starting:

  ✓ .NET 8 SDK         → https://dotnet.microsoft.com/download/dotnet/8.0
                         Verify: dotnet --version   (should show 8.x.x)

  ✓ SQL Server Express  → https://www.microsoft.com/en-us/sql-server/sql-server-downloads
    or SQL Server Developer Edition (free)
    Verify: connect via SSMS or
            sqlcmd -S .\SQLEXPRESS -Q "SELECT @@VERSION"

  ✓ Node.js 18+ (or 16 with Vite 4 as configured)
    Verify: node --version

  ✓ Git (optional but recommended)


=============================================================
 STEP 1 — SET UP THE BACKEND
=============================================================

1a. Open PowerShell / Terminal and navigate to the backend folder:

    cd C:\Users\sai pranay\Desktop\Sai\Portfolio\portfolio-backend


1b. Restore NuGet packages:

    dotnet restore


1c. Set your SQL Server connection string.
    Open:  src\Portfolio.API\appsettings.json

    Find the "ConnectionStrings" section and update:

    "DefaultConnection": "Server=.\\SQLEXPRESS;Database=PortfolioDb;Trusted_Connection=True;TrustServerCertificate=True;"

    Common variations:
    • Named instance:      Server=.\SQLEXPRESS
    • Default instance:    Server=localhost  or  Server=.
    • SQL auth:            Server=.;Database=PortfolioDb;User Id=sa;Password=YOUR_SA_PASSWORD;TrustServerCertificate=True;


1d. Set your Gmail App Password for email sending.
    Open:  src\Portfolio.API\appsettings.json

    Find "EmailSettings" and update the "Password" field:

    "Password": "abcd efgh ijkl mnop"    ← your 16-char Gmail App Password

    How to get a Gmail App Password:
      1. Go to https://myaccount.google.com/security
      2. Enable 2-Step Verification (if not already on)
      3. Go to https://myaccount.google.com/apppasswords
      4. Select App: Mail, Device: Windows Computer → Generate
      5. Copy the 16-character password shown (spaces don't matter)


1e. Create the database and apply EF Core migrations.
    Run from the portfolio-backend root folder:

    dotnet ef migrations add InitialCreate --project src\Portfolio.Infrastructure --startup-project src\Portfolio.API

    Then apply migrations to create the PortfolioDb database:

    dotnet ef database update --project src\Portfolio.Infrastructure --startup-project src\Portfolio.API

    Verify in SSMS: you should see PortfolioDb → Tables → dbo.ContactMessages


1f. Run the API:

    dotnet run --project src\Portfolio.API

    You should see output like:
      info: Microsoft.Hosting.Lifetime[14]
            Now listening on: http://localhost:5000
      info: Microsoft.Hosting.Lifetime[14]
            Now listening on: https://localhost:7000

    Test it is running — open in browser:
      http://localhost:5000/swagger

    You will see the Swagger UI with POST /api/contact and GET /api/contact.


=============================================================
 STEP 2 — SET UP THE REACT FRONTEND
=============================================================

2a. Open a NEW PowerShell window and navigate to the frontend:

    cd C:\Users\sai pranay\Desktop\Sai\Portfolio\portfolio-frontend


2b. Create your .env file (if not done already):

    copy .env.example .env


2c. Open .env in Notepad and confirm:

    VITE_API_BASE_URL=http://localhost:5000

    This must match the port your ASP.NET Core API is running on.


2d. Install dependencies (if not done already):

    npm install


2e. Start the React dev server:

    npm run dev

    Output should show:
      VITE v4.x.x  ready in xxx ms
      ➜  Local:   http://localhost:3000/

=============================================================
 STEP 4 — QUICK TROUBLESHOOTING REFERENCE
=============================================================

Problem: "Cannot connect to SQL Server"
Fix:     Check the connection string.
         Make sure SQL Server service is running:
         Services → SQL Server (SQLEXPRESS) → Start

Problem: "No such host is known" or SMTP error
Fix:     Your Gmail App Password is wrong or missing.
         Double-check appsettings.json "Password" field.

Problem: CORS error in browser console ("blocked by CORS policy")
Fix:     Confirm API is running on http://localhost:5000
         and VITE_API_BASE_URL in .env matches exactly.

Problem: "relation does not exist" or EF migration errors
Fix:     Re-run:
         dotnet ef database update --project src\Portfolio.Infrastructure --startup-project src\Portfolio.API

Problem: React form submits but shows generic error
Fix:     Open browser DevTools → Network tab → inspect the POST /api/contact
         response body. The API returns a JSON error message field.


=============================================================
 PRODUCTION DEPLOYMENT NOTES (Azure)
=============================================================

When you're ready to deploy:

1. Azure SQL: update "DefaultConnection" in appsettings.Production.json
   (or use Azure App Service → Configuration → Connection strings).

2. SMTP credentials: store the Gmail App Password in Azure Key Vault,
   not in appsettings.json. Reference it via Key Vault references in
   App Service configuration.

3. CORS: add your production domain to the WithOrigins() list in Program.cs.

4. Remove the auto-migrate block from Program.cs and run migrations
   manually or via your CI/CD pipeline:
   dotnet ef database update --connection "YOUR_AZURE_SQL_CONNECTION_STRING"

5. Add [Authorize] to GET /api/contact before going live.
